## 2. 데이터 품질 이슈 및 조치

### 2.1 종료사유 오입력 (1건 제거)

- 문제: `종료사유` 값이 '경기도 포천시'인 행 1건 발견. 시도 컬럼 값이 종료사유에 잘못 입력된 것으로 판단.
- 조치: 해당 행 제거 (789,875 → 789,874건)

### 2.2 중성화 여부 오입력 (1건 정제)

- 문제: `중성화 여부` 컬럼에 동물 특징 텍스트가 입력된 행 1건 발견. ("코 분홍에 가운데 점박이. 눈곱. 온순...")
- 조치: 해당 값을 '미상'으로 대체

### 2.3 체중 이상치

- 문제: 극단적 이상치 다수 존재
  - 음수 값: 4건
  - 0kg: 493건
  - 100kg 초과: 407건
  - 최대값: 553,799,387kg (명백한 오입력)
- 조치: 0 이하 또는 200kg 초과 값을 NaN 처리. 200kg 기준은 대형견(그레이트 데인 등)의 극단 체중을 고려한 보수적 기준.

### 2.4 나이 컬럼 비정형 형식

- 문제: 나이가 숫자(나이)가 아닌 출생연도로 기록. 형식도 '2023', '2023(60일미만)' 등 혼재.
- 조치: 파싱 로직 구현 (상세 내용은 3.1절 참조)

### 2.5 보호기간일 이상치

- 문제: 공고종료일 - 공고시작일 계산 시 음수(-3,277일)나 극단값(72,328일) 존재.
- 조치: 파생 컬럼으로 생성만 하고 별도 클리핑 미적용. 분석 시 필터링 필요.

## 3. 피처 엔지니어링 상세

### 3.1 나이 관련 피처 (3개 생성)

**원본 형태 분석**

나이 컬럼은 대부분 출생연도(예: '2019', '2020')로 기록되어 있으며, 60일 미만 어린 동물은 '2023(60일미만)' 형태로 표기.

**변환 로직**

```
입력: 나이(문자열), 접수일자(datetime)
처리:
  1. '(60일미만)' 포함 여부 → 60일미만 플래그 추출
  2. 괄호 제거 후 숫자 추출
  3. 1990~2026 범위 → 출생연도로 판단 → 추정나이 = 접수년도 - 출생연도
  4. 0~30 범위 → 직접 나이 입력으로 판단 → 그대로 사용
  5. 그 외 → NaN
```

**생성 컬럼**

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| 추정나이 | float | 접수년도 - 출생연도 (0 이상) |
| 60일미만 | bool | 60일 미만 여부 플래그 |

## 5. 미처리 사항 및 향후 과제

| 항목 | 설명 | 우선순위 |
|------|------|----------|
| 발견장소 지오코딩 | 주소 → 위경도 변환 시 지도 시각화 가능 | 중 |
| 특징 텍스트 분석 | NLP로 키워드 추출 → 입양률 영향 요인 분석 | 중 |
| 보호기간일 이상치 처리 | 음수/극단값 클리핑 기준 설정 필요 | 하 |
| 동물종류 '기타' 세분화 | 11,115건의 '기타' 품종 추가 분류 가능 | 하 |
| 색상 표준화 | '검정&흰색', '검흰', '흑백' 등 동일 색상 통합 | 하 |
